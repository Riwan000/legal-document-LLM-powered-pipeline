## Findings

### Critical

#### C-01 — `/api/admin/clear-all` has no authentication (`backend/main.py:1512`)

```python
@app.post("/api/admin/clear-all")
async def clear_all_documents():
```

The endpoint accepts any unauthenticated POST and permanently deletes every document, embedding, clause, and on-disk file in the system. No auth header, API key, or admin token is checked. Any HTTP client that can reach port 8000 (directly or through a reverse proxy that doesn't strip paths) can invoke it.

This is acceptable in a local dev setup but must be guarded before any shared or production deployment.

**Recommendation:** Add a dependency that validates an `X-Admin-Token` header against an env var:

```python
from fastapi import Header, Depends, HTTPException

def require_admin_token(x_admin_token: str = Header(default="")):
    if x_admin_token != os.getenv("ADMIN_TOKEN", ""):
        raise HTTPException(status_code=403, detail="Forbidden")

@app.post("/api/admin/clear-all", dependencies=[Depends(require_admin_token)])
```

### High

#### H-01 — `_is_likely_operative_contract` now rejects MSA and NDA contracts (`contract_review_service.py:436`)

The H-02 fix changed `and` → `or`:

```python
# After fix
if "agreement" not in lead or ("employer" not in lead and "employee" not in lead):
    return False
```

This now requires both: the lead must contain `"agreement"` **AND** must contain either `"employer"` or `"employee"`. A standard NDA (parties: "Disclosing Party" / "Receiving Party") or MSA (parties: "Service Provider" / "Client") satisfies the `"agreement"` condition but not the employer/employee condition — the function returns `False`, triggering `document_classification_warning` for every non-employment contract.

The original `and` was indeed too permissive (the heuristic rarely fired), but the `or` overcorrects. Both NDA and MSA review flows would now always show the "document may not be operative" warning.

**Recommendation:** Expand the party-signal check to cover all contract types:

```python
_PARTY_SIGNALS = ("employer", "employee", "party", "service provider",
                   "disclosing party", "licensor", "licensee", "contractor")
if "agreement" not in lead and not any(p in lead for p in _PARTY_SIGNALS):
    return False
```

Or simply remove this guard and rely on the more robust `DOC_CLASSIFY_OPERATIVE_PHRASES` + academic-density checks below it.

#### H-02 — `get_stats()` now triggers FAISS index load on every sidebar render (`vector_store.py:665`)

The prior intentional design was removed:

```python
# BEFORE — intentional comment explaining the no-load design
# NOTE: We intentionally do NOT call `_ensure_loaded()` here so that
# lightweight health/stats checks do not trigger disk I/O. Stats will
# reflect the in-memory view (which may be empty) until the first
# real search/ingest operation loads persisted data.
```

`_ensure_loaded()` was added. The Streamlit sidebar DB panel calls `get_vector_stats()` → `GET /api/stats` → `get_stats()` → `_ensure_loaded()` on every page render, including pages that never use the vector store. A large FAISS index (potentially 100 MB+) will be loaded from disk on the first stats-only call before any actual search is performed.

**Recommendation:** Cache the sidebar stats in `st.session_state` with a short TTL so repeated renders don't re-request:

```python
_STATS_CACHE_KEY = "_db_stats_cache"
_STATS_TTL = 30  # seconds

def get_vector_stats() -> dict:
    import time
    cache = st.session_state.get(_STATS_CACHE_KEY)
    if cache and time.time() - cache["ts"] < _STATS_TTL:
        return cache["data"]
    try:
        data = requests.get(f"{API_BASE_URL}/api/stats", timeout=10).json().get("vector_store", {})
    except Exception:
        data = {}
    st.session_state[_STATS_CACHE_KEY] = {"ts": time.time(), "data": data}
    return data
```

#### H-03 — Sidebar DB panel makes HTTP calls at module scope on every page render (`frontend/app.py`, sidebar block)

```python
with st.sidebar.expander("🗄️ Database", expanded=False):
    stats = get_vector_stats()   # HTTP call
    ...
    docs = get_documents()       # HTTP call
```

These calls execute unconditionally on every Streamlit page render — regardless of which page the user is on. When the backend is starting up or slow, every page load stalls waiting for both responses. `get_vector_stats()` silently returns `{}` on failure (invisible to the user), while `get_documents()` now shows a `st.warning`. The inconsistency and the render-loop placement are both issues.

**Recommendation:** Move the calls inside a `if st.sidebar.button("🔄 Refresh")` handler, or gate them on a session-state timestamp so they run at most once per minute.

### Medium

#### M-01 — `classify_clause_type` computes `string_scores` in dead-code branch (`clause_taxonomy.py:195–205`)

```python
# Fall back to string-keyed keywords
string_scores: Dict[str, int] = {}
for slug, keywords in self.string_type_keywords.items():
    score = sum(1 for kw in keywords if kw in text_lower)
    if score > 0:
        string_scores[slug] = score

if string_scores:
    return ClauseType.OTHER   # ← identical to the next line
return ClauseType.OTHER
```

The `for` loop and `if string_scores` block are dead code: both branches return `ClauseType.OTHER`. The comment says "caller can read `normalized_clause_type`" but no attribute is set here. The entire loop is wasted CPU on every call that falls through enum scoring.

**Recommendation:** Remove the dead loop or add the intended `self._last_string_slug = max(...)` assignment so the caller can actually read it.

#### M-02 — `_identify_contradiction_risks` redefines helper functions on every invocation (`contract_review_service.py:1140–1160`)

`_extract_duration` and `_evidence_text_for` are nested functions recreated on every call to `_identify_contradiction_risks`. Each call allocates new function objects and closures. The design also makes these helpers invisible to unit tests.

**Recommendation:** Promote both to private module-level functions (`_extract_duration`, `_evidence_text_for`).

#### M-03 — `clear_all_documents` returns `200 OK` with `"status": "cleared"` on partial failure (`main.py:1536–1578`)

Store-deletion errors are caught, logged, and silently ignored. The response always returns `{"status": "cleared"}` even if half the stores failed. The `total_chunks` / `total_clauses` counters also default to 0 if the store raised before incrementing, making the counts unreliable.

**Recommendation:** Collect errors and reflect them in the response:

```python
errors = []
# inside each except block:
errors.append(f"{doc_id}: {e}")
# in return:
return {"status": "partial" if errors else "cleared", "warnings": errors, ...}
```

#### M-04 — `get_vector_stats` silently ignores exceptions (`frontend/app.py`)

```python
except Exception:
    return {}
```

`get_documents()` was correctly updated to show `st.warning(f"Could not load documents: {e}")`. `get_vector_stats()` still swallows exceptions, showing `0` in all metric widgets rather than a diagnostic message. The inconsistency makes backend connectivity failures invisible in the DB stats panel.

**Recommendation:** Apply the same pattern:
```python
except Exception as e:
    st.warning(f"Could not load vector stats: {e}")
    return {}
```

#### M-05 — `statutory_notes` may serialize as `{}` instead of `null` (`contract_review_service.py:~2075`)

```python
if jur_data:
    statutory_notes = {
        EXPECTED_CLAUSE_DISPLAY_NAMES.get(k, k): v
        for k, v in jur_data.items()
        if k in expected           # ← could filter to empty dict
    }
```

If `jur_data` is non-empty but no key matches `expected` (e.g. a jurisdiction YAML with only `non_compete` notes reviewed against an NDA profile that has no `non_compete`), `statutory_notes` becomes `{}` instead of `None`. The response model field is `Optional[Dict[str, Any]]` with `default=None`. The serialized JSON will contain `"statutory_notes": {}` rather than `null`, which is inconsistent with the field's design contract.

**Recommendation:** Guard the assignment:
```python
_notes = {EXPECTED_CLAUSE_DISPLAY_NAMES.get(k, k): v for k, v in jur_data.items() if k in expected}
statutory_notes = _notes or None
```

### Low

#### L-01 — `_resolve_clause_display_name` produces lowercase display names for multi-word IDs (`contract_review_service.py:385–391`)

With the L-02 fix, `" ".join(parts[1:-1])` produces `"non compete clause"` (all lowercase segments from a snake_case ID). The camelCase splitter `re.sub(r"([a-z])([A-Z])", ...)` is a no-op on already-lowercase text, so the result returns as `"non compete clause"` rather than `"Non Compete Clause"`.

**Recommendation:** Apply `.title()` before returning `cleaned`:
```python
return cleaned.strip().title()
```

#### L-02 — `_extract_duration` 30-day month conversion may mismatch "3 months" vs "90 days" (`contract_review_service.py:1147`)

The ratio threshold is 1.5×. Three months = 90 days; if both clauses say 90 days, ratio = 1 (no flag). If one says "3 months" (= 90 days) and the other says "61 days", ratio ≈ 1.48 — no flag, despite real difference. At the current threshold the main practical risk is false negatives on near-borderline mismatches, which is benign.

#### L-03 — `vector_store.delete_document` return value may be `None` (`main.py:1542`)

```python
total_chunks += vector_store.delete_document(doc_id)
```

If `delete_document` returns `None` for a document not found (depending on the implementation), this line raises `TypeError: unsupported operand type(s) for +=: 'int' and 'NoneType'`. The outer `try/except` would catch it, but `total_chunks` would not be incremented. Verify `delete_document` always returns `int`.

#### L-04 — Bidirectional heading-match check narrowed (`contract_review_service.py:~1871`)

The diff shows:
```python
# Before
if norm_heading_kw in title_norm or title_norm in norm_heading_kw:
# After
if norm_heading_kw in title_norm:
```

The removed `title_norm in norm_heading_kw` direction handled cases where the document heading is shorter than the synonym keyword (e.g., heading `"non-compete"` is a substring of keyword `"non compete agreement"`). Removing it means clauses with abbreviated headings may no longer get implicit heading-match detection. Assess whether this caused false negatives in practice.

#### L-05 — `contradiction_risks` presence in response not verified in PDF generation

`ContractReviewResponse.contradiction_risks` is populated and included in the API response, but the PDF report builder (presumably in `_build_pdf` or similar) may not render these items. Verify that contradiction risk items appear in the exported PDF under a "Contradiction Warnings" section or within the risk table.
