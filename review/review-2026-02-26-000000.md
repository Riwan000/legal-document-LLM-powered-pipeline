## CRITICAL

### C-01 — Infinite recursion in `rag_service.py`
- **Category:** Logic
- **Severity:** Critical
- **Location:** `backend/services/rag_service.py` ~line 199
- **Description:** `_search_without_translation` calls itself unconditionally. When `ENABLE_QUERY_TRANSLATION=True` and a translation service is present, every call to `search()` that reaches the translation path will recurse until Python raises a `RecursionError`. The method's docstring says "without translation logic" but its body re-enters the translation block.
- **Fix:** Extract the raw embedding + vector-store call into a true base method (`_search_core`) and call that from both paths.

### C-02 — Wildcard CORS with `allow_credentials=True` (`main.py:103`)
- **Category:** Security
- **Severity:** Critical
- **Location:** `backend/main.py:103`
- **Description:** `allow_origins=["*"]` combined with `allow_credentials=True` is both a browser CORS spec violation and a security risk for a system that processes confidential legal documents.
- **Fix:** Load allowed origins from config/env; default to `["http://localhost:8501"]` for development.

## HIGH

### H-01 — Pickle deserialization of untrusted files (`vector_store.py:604`)
- **Category:** Security
- **Severity:** High
- **Location:** `backend/services/vector_store.py:604`
- **Description:** Both the metadata file (`.metadata.pkl`) and the numpy index file are loaded with `pickle.load` — a known arbitrary code execution vector if the files can be replaced by a malicious actor.
- **Fix:** Serialize metadata as JSON; use `numpy.save`/`numpy.load(allow_pickle=False)` for vectors.

### H-02 — Debug artifacts + relative audit log path (`chat_orchestrator.py`)
- **Category:** Maintainability / Logic
- **Severity:** High
- **Location:** `backend/services/chat_orchestrator.py:53, 248–278, 641–691`
- **Description:** The audit log uses a CWD-relative `Path("logs/chat.log")`. Twelve debug write blocks append to `.cursor/debug.log` on every chat turn (a Cursor IDE directory). Both break in production deployments.
- **Fix:** Anchor log paths to `Path(__file__).resolve().parent.parent / "logs"`; remove all `#region agent log` debug blocks.

### H-03 — Module-level debug logger with per-call file open (`evidence_guardrail_service.py:38–47`)
- **Category:** Performance / Maintainability
- **Severity:** High
- **Location:** `backend/services/evidence_guardrail_service.py:38–47`
- **Description:** `_dbg()` opens and appends to `.cursor/debug.log` on every guardrail check, causing O(N) file I/O per conversation turn. In multi-user load this is a measurable performance issue.
- **Fix:** Remove the `_dbg` function and all call sites; use `logging.debug()` instead.

### H-04 — Synchronous LLM summarization in the chat hot path (`session_manager.py:258–277`)
- **Category:** Performance / Logic
- **Severity:** High
- **Location:** `backend/services/session_manager.py:258–277`
- **Description:** `enforce_limits()` calls `summarizer.summarize(session)` inline — a blocking Ollama LLM call — causing a multi-second stall on every `MAX_SESSION_TURNS`-th message. The user's HTTP request is blocked for the full summarization duration.
- **Fix:** Offload to a background task (e.g. `asyncio.create_task` or FastAPI `BackgroundTasks`); do not block the user's response pipeline.

### H-05 — Over-broad "chunk/token/embedding" guard silently drops clauses (`structured_clause_extraction.py:298–302`)
- **Category:** Logic / Edge Case
- **Severity:** High
- **Location:** `backend/services/structured_clause_extraction.py:298–302`
- **Description:** Any document page whose text contains the words "token", "chunk", or "embedding" (all legitimate in IP agreements, data-processing addenda, and fintech contracts) causes the entire document to return zero clauses with a silent `return []`. The check was likely intended to filter pipeline debug output leaked into the document.
- **Fix:** Remove this heuristic or make it far more specific (e.g. check only the first 50 chars for a very specific debug marker).

## MEDIUM

### M-01 — `_normalize` double-substitution bug (`query_classifier.py:107`)
- **Category:** Logic
- **Severity:** Medium
- **Location:** `backend/services/query_classifier.py:107`
- **Description:** `_normalize` uses bare `str.replace` in a loop; "termination" → "terminate" is applied before "determination" is checked, so "determination" becomes "deteterminate".
- **Fix:** Sort replacements longest-first, or use a compiled regex with word boundaries.

### M-02 — Incompatible `_chunk_key` formats break deduplication (`retrieval_router.py:82`, `chat_orchestrator.py:152`)
- **Category:** Logic
- **Severity:** Medium
- **Location:** `backend/services/retrieval_router.py:82`, `backend/services/chat_orchestrator.py:152`
- **Description:** The two components format chunk deduplication keys differently (`doc_id::chunk_id` vs `doc_id|chunk_id`), so the same chunk is never recognized as a duplicate across the pipeline boundary.
- **Fix:** Centralize the key format in a shared utility or constant.

### M-03 — Dead branch in evidence score logic (`evidence_guardrail_service.py:173`)
- **Category:** Logic
- **Severity:** Medium
- **Location:** `backend/services/evidence_guardrail_service.py:173`
- **Description:** Lines 176 and 177 cover the same condition; the second branch is unreachable.
- **Fix:** Remove the duplicate branch.

### M-04 — Zero-chunk refusal passes through unvalidated LLM answer (`chat_orchestrator.py:693`)
- **Category:** Logic / Edge Case
- **Severity:** Medium
- **Location:** `backend/services/chat_orchestrator.py:693`
- **Description:** When the retrieval returns zero chunks, the code enters a refusal branch but then falls through via `pass`, potentially allowing a hallucinated LLM answer to reach the user unmarked.
- **Fix:** Ensure the refusal always returns a clearly-marked "no evidence" response; replace `pass` with an explicit `return`.

### M-05 — Non-atomic `add_chunks` in vector store (`vector_store.py:177`)
- **Category:** Edge Case
- **Severity:** Medium
- **Location:** `backend/services/vector_store.py:177`
- **Description:** If an exception occurs between `index.add` and `metadata.append`, the FAISS index and the metadata list are permanently out of sync, causing ghost entries or index-out-of-range errors on subsequent queries.
- **Fix:** Buffer additions and commit both structures atomically under a lock; or catch the exception and roll back the index addition.

### M-06 — `_JURISDICTION_PATTERN` too narrow (`session_manager.py:43`)
- **Category:** Logic / Edge Case
- **Severity:** Medium
- **Location:** `backend/services/session_manager.py:43`
- **Description:** Common GCC contract phrasings like "subject to laws of [jurisdiction]" or "governed by [country] law" never match the current regex, so jurisdiction extraction silently fails for a large class of real-world contracts.
- **Fix:** Expand the pattern to include these common variants.

### M-07 — `risk_explanations.yaml` loaded from CWD-relative path (`contract_review_service.py:1049`)
- **Category:** Logic
- **Severity:** Medium
- **Location:** `backend/services/contract_review_service.py:1049`
- **Description:** The file is opened with a bare relative path, which resolves relative to the process working directory. In Docker or CI this is almost certainly wrong, silently returning an empty dict and suppressing all risk explanations.
- **Fix:** Use `Path(__file__).resolve().parent / "risk_explanations.yaml"` or load from config.

### M-08 — `run()` method is 568 lines, untestable in isolation (`contract_review_service.py:1353`)
- **Category:** Complexity / Maintainability
- **Severity:** Medium
- **Location:** `backend/services/contract_review_service.py:1353`
- **Description:** The `run()` method is 568 lines with 10 sequential pipeline steps and 7 inline post-processing passes. This makes it impossible to unit-test individual steps, difficult to debug failures, and fragile under change.
- **Fix:** Extract each named step into a private method; coordinate them from `run()` with a clear pipeline loop.

### M-09 — `detect_document_type` over-classifies as CONTRACT (`structured_clause_extraction.py:892`)
- **Category:** Logic / Assumption
- **Severity:** Medium
- **Location:** `backend/services/structured_clause_extraction.py:892`
- **Description:** The heuristic returns `CONTRACT` for any document containing the word "party" — too broad; advisory opinions, court rulings, and term sheets all use this word.
- **Fix:** Require at least 2 corroborating signals (e.g. "party" + "agreement" + "obligation") before classifying as CONTRACT.

### M-10 — "law" in jurisdiction keyword list suppresses clarification for nearly every legal query (`query_classifier.py:281`)
- **Category:** Logic
- **Severity:** Medium
- **Location:** `backend/services/query_classifier.py:281`
- **Description:** The string "law" appears in the jurisdiction keyword list. Almost every legal query contains "law", so the classifier labels nearly every query as jurisdiction-specific, suppressing the clarification prompt and misrouting retrieval.
- **Fix:** Remove "law" from the list; replace with more discriminating terms like "jurisdiction", "governing law clause", "choice of law".

### M-11 — Duplicate risk patterns cause double risk items (`contract_review_service.py:41`)
- **Category:** Logic
- **Severity:** Medium
- **Location:** `backend/services/contract_review_service.py:41`
- **Description:** `"probation"` and `"probation_period"` are both present as separate patterns; since "probation_period" always matches "probation", the same text generates two separate risk items with different keys.
- **Fix:** Deduplicate patterns; keep only the more specific one.

### M-12 — Prompt injection strip easily bypassed (`chat_orchestrator.py:73`)
- **Category:** Security
- **Severity:** Medium
- **Location:** `backend/services/chat_orchestrator.py:73`
- **Description:** The injection-stripping regex looks for specific delimiters but is bypassed by minor variations (Unicode look-alikes, extra whitespace, mixed casing). A legal document could contain an adversarial payload that reaches the LLM.
- **Fix:** Use a whitelist approach: only pass known-safe fields into the prompt template; treat all document-sourced text as untrusted.

### M-13 — `print()` used for OCR errors instead of `logging` (`file_parser.py:73–80`)
- **Category:** Maintainability
- **Severity:** Medium
- **Location:** `backend/utils/file_parser.py:73–80`
- **Description:** OCR errors are printed to stdout, bypassing the application's log handlers. In production, these errors are invisible to log aggregators.
- **Fix:** Replace `print()` with `logging.warning()` or `logger.warning()`.

## Positive Aspects

- **JSON audit trail** in `chat_orchestrator.py` provides full turn reconstructability for debugging.
- **Two-pass guardrail** (weak-evidence check → regenerate with higher threshold) in `evidence_guardrail_service.py` is a robust pattern.
- **Lazy vector store loading** avoids startup latency and memory waste when the store hasn't been built.
- **Min-max score normalization** in dual retrieval fusion prevents any single engine from dominating by scale.
- **Fail-closed extraction** — `structured_clause_extraction.py` returns `[]` on every error path rather than partial results.
- **YAML contract profile system** (`contract_profiles/`) is cleanly extensible for new document types without code changes.
- **`established_facts` injection** in `chat_orchestrator.py` provides cross-turn consistency for document type, jurisdiction, and parties.
- **Page-level fallback scan** for Schedules/Annexures prevents SLA and commercial schedule clauses from being silently dropped.
