## Findings

### CRITICAL — Logic bugs that produce wrong output

---

#### [C1] Levenshtein distance tolerance is too loose
**File:** `backend/services/contract_review_service.py` ~line 803
**Category:** Logic

```python
TERMINATION_TOKEN_DISTANCE = 2

def _looks_like_terminate(token: str) -> bool:
    return _levenshtein_distance(token, "terminate") <= TERMINATION_TOKEN_DISTANCE
```

A distance of 2 allows words like "terminus", "temporal", "tempered" to match "terminate". This can cause false-positive termination clause detections in documents that discuss time periods or insurance.

**Fix:** Tighten to distance ≤ 1, or add a length-proximity guard:
```python
def _looks_like_terminate(token: str) -> bool:
    if abs(len(token) - len("terminate")) > 1:
        return False
    return _levenshtein_distance(token, "terminate") <= 1
```

---

#### [C2] Duplicate categories in taxonomy cause indeterminate classification
**File:** `backend/services/clause_taxonomy.py` ~lines 35–38
**Category:** Logic / Edge Cases

`_LEGAL_CATEGORY_PRIORITY` contains both `"probation"` and `"probation_period"`, and both `"conduct"` and `"conduct_discipline"`. The corresponding `_CATEGORY_SLUG_KEYWORDS` maps partially overlapping keyword sets to both names. When a clause contains "probationary period", both categories score equally, and the winner depends entirely on list order — making results unpredictable across Python version or dict ordering changes.

**Fix:** Remove the shorter alias (`"probation"`, `"conduct"`) from both `_LEGAL_CATEGORY_PRIORITY` and `_CATEGORY_SLUG_KEYWORDS`, keeping only the compound forms. Update any downstream string comparisons accordingly.

---

#### [C3] Arabic numeral clause detection misses colon delimiter
**File:** `backend/services/structured_clause_extraction.py` ~line 653
**Category:** Edge Cases / GCC support

```python
match = re.match(r'^([٠-٩]+)[\.\)]\s*(.+)', line_stripped)
```

Arabic-numbered clauses in GCC contracts frequently use a colon delimiter (`١: التعريفات`). This regex only handles `.` and `)`, so those clause starts are never detected, and the content falls through to page-level fallback chunks.

**Fix:**
```python
match = re.match(r'^([٠-٩]+)[\.\):]\\s*(.+)', line_stripped)
```

### MEDIUM — Design issues that degrade output quality

---

#### [M1] structure_class downgrade happens after the implicit-logic flag is set
**File:** `backend/services/contract_review_service.py` ~lines 2019–2035
**Category:** Logic / Ordering

`section_non_standard` blocks are downgraded to `"provision"` after `used_implicit_or_distributed_logic` is computed (line 2035). This means the flag is set based on the pre-downgrade class, but the blocks displayed in the UI show the post-downgrade class. The UI disclaimer fires in cases where no `section_non_standard` blocks are ultimately shown to the user.

**Fix:** Move the downgrade loop to before line 2028 (before the implicit-logic flag computation).

---

#### [M2] No deduplication of RiskItems by description
**File:** `backend/services/contract_review_service.py` ~line 1700 (risk aggregation)
**Category:** Complexity / Output quality

Problematic-language scans and missing-clause checks can emit multiple `RiskItem`s with identical `description` strings (e.g., the same ambiguous phrase flagged in two adjacent clauses). The risk table then shows duplicate rows.

**Fix:** Add a dedup pass after risk aggregation, grouping by `(description, frozenset(clause_ids))` and merging `page_numbers`/`verbatim_evidence`.

---

#### [M3] `"confidential"` keyword in ADMINISTRATIVE_MATERIAL section patterns
**File:** `backend/services/structured_clause_extraction.py` ~lines 166–190
**Category:** Edge Cases

The ADMINISTRATIVE_MATERIAL pattern list includes `confidential` as a standalone keyword. If an NDA's first operative page begins with "CONFIDENTIAL INFORMATION AGREEMENT", the page classifier may score it as administrative rather than contractual, silently dropping its clauses.

**Fix:** Replace the bare `confidential` match with a more specific pattern that requires it alongside `internal use only` or `private and confidential` (the headers stamped on cover sheets), not standalone.

---

#### [M4] YAML clause keywords loaded at module import with silent failure
**File:** `backend/services/clause_taxonomy.py` ~line 99
**Category:** Maintainability / Edge Cases

```python
_CLAUSE_KEYWORDS: Dict[str, List[str]] = _load_clause_keywords()
```

If `clause_keywords.yaml` is missing or malformed, `_load_clause_keywords()` logs a warning and returns `{}`. The module continues to load, but all string-type clause classifications (`ip_ownership`, `sla_obligations`, etc.) silently fail — returning no match instead of an error.

**Fix:** Validate the returned dict is non-empty (or has expected keys) after load. Raise at startup if the YAML is absent, rather than silently degrading.

---

### LOW — Minor issues

---

#### [L1] Frontend: no progress indicator during 300-second contract review
**File:** `frontend/app.py` ~line 326
**Category:** UX

```python
response = requests.post(f"{API_BASE_URL}/api/contract-review", data=data, timeout=300)
```

The UI is fully blocked for up to 5 minutes with no visual feedback. Streamlit's `st.spinner` or a status message should wrap this call.

---

#### [L2] Frontend: `response.json()` not guarded against non-JSON error bodies
**File:** `frontend/app.py` ~lines 332–335
**Category:** Edge Cases

```python
err = e.response.json()
st.error(f"Contract review failed: {err.get('detail', e.response.text)}")
```

If the backend returns a plain-text 5xx (e.g., a gateway timeout), `response.json()` raises `ValueError` and the error message is lost. Wrap in `try/except ValueError`.

## Positive Notes

- **Models layer is clean.** All Pydantic models are well-typed, match the PRD, and have no missing fields.
- **Fail-closed defaults throughout.** Extraction, classification, and detection all prefer conservative false-negatives over false-positives.
- **Evidence-backed risks only.** Every `RiskItem` is grounded in actual `VerbatimSnippet` evidence — no hallucinated risk items.
- **Good separation of concerns.** Models → Service → API → UI layers are clean; no business logic in the endpoint.
- **G1–G9 post-passes are comprehensive.** Implicit governing law, distributed benefits, heading synonyms, LLM binary resolver, and contradiction detection all work together correctly.
- **Keyword YAML externalization (Group 1a).** `clause_keywords.yaml` makes taxonomy tuning accessible without touching Python code.
- **GCC jurisdiction awareness.** `jurisdiction_statutes.yaml` and jurisdiction-aware risk checks are a strong feature for the target market.
